# План реализации Frontend-модуля и системы кодогенерации

**Обоснование и брейншторм:** [brainstorm-frontend.md](../brainstorms/brainstorm-frontend.md)

Настоящий план описывает итеративное внедрение нового подхода к фронтенду и кодогенерации на базе Astro, React, Refine и OpenAPI. В плане опущены рассуждения и альтернативы — оставлены только конкретные шаги и критерии успешности.

## Фаза 1: Генератор роутеров и access control
**Цель:** Добавить write-once генерацию роутеров с декларативным управлением доступом. `SchemasGenerator` остаётся без изменений — генерирует и ORM-модели, и Pydantic-схемы (Create/Read/Update) как раньше.

**Задачи:**
- [ ] Реализовать `RouterStubGenerator` (write-once генерация роутеров с базовым CRUD и TODO-маркерами бизнес-логики). Роутеры импортируют Pydantic-схемы из `shared.shared.generated.schemas`.
- [ ] Добавить поле `access` в доменные спеки (`services/*/spec/*.yaml`): значения `public`, `user`, `admin`, `custom`. Реализовать маппинг в вызовы `Depends()` в `RouterStubGenerator`.
- [ ] Добавить базовые dependency-функции (`require_authenticated_user`, `require_admin`) в шаблон API-приложения.
- [ ] Написать unit-тесты для `RouterStubGenerator`.

**Критерии приёмки:**
- Генерация работает без ошибок, создаются новые стабы без перезаписи внесённых ранее пользовательских изменений.
- Запуск тестов шаблона проходит успешно.
- В сгенерированных роутах присутствуют корректные зависимости авторизации (`Depends`) согласно доменным спекам.

## Фаза 2: Пайплайн OpenAPI и TypeScript
**Цель:** Автоматизировать синхронизацию API-контрактов между бэкендом и фронтендом без необходимости поднятия всего стека (БД, брокеры и т.д.).

**Задачи:**
- [ ] Написать скрипт статического дампа OpenAPI (`scripts/dump_openapi.py`), который извлекает схему `openapi.json` из инстанса FastAPI-приложения на этапе импорта.
- [ ] Настроить кодогенерацию TypeScript-интерфейсов из `openapi.json` с помощью `openapi-typescript` в директорию `frontend/src/generated/types.ts`.
- [ ] Добавить статический базовый `apiClient.ts` (fetch-обёртка) в scaffold-генерацию фронтенда.
- [ ] Настроить хук `pre-push` и проверки CI для контроля актуальности файла `openapi.json` и сгенерированных TypeScript-типов.

**Критерии приёмки:**
- Команда `make generate` успешно выгружает схему и генерирует `.ts` типы.
- Ошибка инициализации БД (сайд-эффекты) на глобальном уровне в роутерах роняет скрипт генерации дампа, что обеспечивает защиту архитектуры.
- Хук `pre-push` не пропускает push с рассинхронизированными API-схемами.

## Фаза 3: UI/UX и базовая инфраструктура Фронтенда
**Цель:** Настроить визуальную основу (Astro + React), UI Kit и глобальный flow обработки ошибок.

**Задачи:**
- [ ] Инициализировать Astro + React. Настроить базовый file-based routing в директории `src/pages`.
- [ ] Интегрировать `shadcn/ui` и подключить дизайн-токены/переменные.
- [ ] Добавить базовый `ThemeContext` для управления светлой/темной темой через `useState`.
- [ ] Настроить глобальный Error Boundary компонент (`<ErrorFallback>`).
- [ ] Внедрить перехваты типовых HTTP-ошибок: 500/403 (перехват ErrorBoundary), 422 (структурированные inline-ошибки под полями форм), отображение toast-нотификаций при мутациях.
- [ ] Добавить фронтенд-пути в `_skip_if_exists` в `copier.yml`, чтобы `copier update` не перезаписывал пользовательские файлы (apiClient, ThemeContext, компоненты и т.д.).

**Критерии приёмки:**
- Фронтенд успешно рендерится с изолированными Astro islands и компонентами shadcn/ui.
- Переключение тем работает корректно и сохраняет состояние.
- Ошибки наглядно отображаются пользователю (уведомления и страницы-заглушки) без краша всего приложения (Astro islands изолируют падения).

## Фаза 4: Система авторизации
**Цель:** Внедрить безопасный flow авторизации (access в памяти + httpOnly refresh токен) поверх UI-инфраструктуры из Фазы 3.

**Задачи:**
- [ ] Разработать бэкенд-эндпоинты: `POST /auth/login` (возвращает access-токен, ставит refresh-токен в cookie), `POST /auth/refresh` (ротация токенов), `POST /auth/logout` (очистка cookie).
- [ ] Реализовать логику в `apiClient.ts`: автоматический interceptor на HTTP `401 Unauthorized` для вызова `/refresh` с promise-lock-механизмом (от дублей запросов).
- [ ] Реализовать React-логику: `AuthContext` для хранения сессии (вызов `/refresh` при mount) и компонент `<ProtectedRoute>` для управления ролями (Public, User, Admin).
- [ ] Добавить `frontend_auth` в `copier.yml`, обернуть auth-файлы в условия Copier (`{% if frontend_auth %}`), позволяющие генерировать лендинг-версию (без auth-слоя) или полноценное приложение.

**Критерии приёмки:**
- Поддерживается успешный логин с получением access-токена в память и refresh-токена в httpOnly cookie.
- Настроен автоматический прозрачный рефреш access-токена по истечении его срока действия.
- При генерации проекта с флагом `frontend_auth: false` код авторизации отсутствует, генерируется облегченный каркас приложения-лендинга.

## Фаза 5: Интеграция Refine (Админка)
**Цель:** Подключить headless-фреймворк Refine для быстрого создания CRUD-интерфейсов администратора.

**Задачи:**
- [ ] Настроить catch-all роут в Astro `src/pages/admin/[...slug].astro` (с директивой `client:only="react"`).
- [ ] Подключить ядро Refine (`@refinedev/core`) и официальный пакет интеграции для `shadcn/ui`.
- [ ] Написать адаптер `authProvider` для Refine, который проксирует вызовы в созданный ранее `AuthContext`.
- [ ] Реализовать статический Refine `dataProvider` в scaffold-шаблоне: универсальный маппинг CRUD на REST-конвенции (`GET /resource`, `POST /resource` и т.д.) через `apiClient`. Ресурсы задаются конфигом, генератор не нужен.
- [ ] Документировать правило использования кастомных экшенов: CRUD-операции идут через Refine, бизнес-действия идут напрямую через `apiClient` в обход `dataProvider`.

**Критерии приёмки:**
- Панель администратора доступна по пути `/admin` исключительно для пользователей с ролью `admin`.
- Ресурсы и таблицы Refine корректно мапятся на эндпоинты бэкенда посредством `dataProvider`.
- Внешний вид интерфейса администратора консистентен с остальным приложением (используется корневая тема shadcn).
